# Security Policy

AgentTrust OS is a development and review tool. It does not guarantee that an AI agent, automation, or application is safe, secure, compliant, or production-ready.

## Reporting Security Issues

Please do not open public issues for vulnerabilities that expose sensitive data, secret-handling bugs, or exploitable behavior.

Report privately by emailing:

abid.redpro@gmail.com

Include:

- affected version or commit
- reproduction steps
- expected impact
- whether the issue involves secrets, private data, or unsafe tool execution

## Safe Usage

- Do not put real API keys in `agenttrust.yaml`.
- Do not commit private customer data in reports.
- Use mock data when sharing examples.
- Treat generated reports as sensitive when they describe internal tools or permissions.
- Keep human approval in place for critical write actions.
